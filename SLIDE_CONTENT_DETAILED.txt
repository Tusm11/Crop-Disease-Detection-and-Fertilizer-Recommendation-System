SLIDE 7: CNN MODEL ARCHITECTURE
================================================================================

TITLE: Deep Learning for Disease Detection

SECTION 1: NETWORK ARCHITECTURE
[Show layer-by-layer breakdown]

Input Layer: 64×64×3 RGB Images
    ↓
Data Augmentation Layer
    ↓
Convolutional Block 1:
  - Conv2D(32 filters, 3×3 kernel) + ReLU
  - Conv2D(32 filters, 3×3 kernel) + ReLU
  - MaxPooling2D(2×2)
  - Dropout(0.25)
    ↓
Convolutional Block 2:
  - Conv2D(64 filters, 3×3 kernel) + ReLU
  - Conv2D(64 filters, 3×3 kernel) + ReLU
  - MaxPooling2D(2×2)
  - Dropout(0.25)
    ↓
Convolutional Block 3:
  - Conv2D(128 filters, 3×3 kernel) + ReLU
  - Conv2D(128 filters, 3×3 kernel) + ReLU
  - MaxPooling2D(2×2)
  - Dropout(0.25)
    ↓
Global Average Pooling
    ↓
Dense(256) + ReLU + Dropout(0.5)
    ↓
Dense(128) + ReLU + Dropout(0.5)
    ↓
Output Layer: Dense(59) + Softmax

SECTION 2: TRAINING CONFIGURATION
- Optimizer: Adam (learning rate = 0.001)
- Loss Function: Sparse Categorical Crossentropy
- Epochs: 25 (with early stopping)
- Batch Size: 32
- Validation Split: 15%

VISUAL ELEMENTS:
- Vertical flowchart showing layer progression
- Color-code different layer types (Conv=blue, Pool=green, Dense=orange)
- Show filter sizes and dimensions at each layer
- Add legend explaining layer types

SPEAKER NOTES:
"Our CNN model has 3 convolutional blocks, each with 2 convolutional layers followed by max pooling and dropout. The convolutional layers extract features from the image, max pooling reduces dimensionality, and dropout prevents overfitting. After the convolutional blocks, we have global average pooling followed by two dense layers. The final output layer has 59 neurons, one for each crop-disease combination. We use Adam optimizer and train for 25 epochs with early stopping to prevent overfitting."

================================================================================

TITLE: Why This Fertilizer? SHAP Explains!

SECTION 1: SHAP ANALYSIS EXAMPLE
Scenario: Wheat crop on Sandy soil

Recommended Fertilizer: UREA

Top 3 Contributing Factors:

   Interpretation: Low nitrogen detected → Need nitrogen-rich fertilizer

2. ⬆️ Crop Type = Wheat
   SHAP Value: +0.35 (Positive impact)
   Interpretation: Wheat is nitrogen-demanding crop

3. ⬆️ Soil Type = Sandy
   SHAP Value: +0.28 (Positive impact)
   Interpretation: Sandy soil drains nutrients quickly

SPEAKER NOTES:
"SHAP (SHapley Additive exPlanations) is a method that explains individual predictions. For example, if our model recommends Urea for a wheat crop on sandy soil, SHAP tells us why. It shows that the low nitrogen level is the main reason (negative impact), the wheat crop type is a secondary reason (positive impact), and the sandy soil is a tertiary reason (positive impact). This transparency is crucial because farmers need to understand why they should use a particular fertilizer. It builds trust and helps them make informed decisions."

- Pandas (Data manipulation)
- NumPy (Numerical computing)
- StandardScaler (Feature scaling)
- LabelEncoder (Categorical encoding)

Visualization:
- Matplotlib (Charts and plots)
- Seaborn (Statistical visualization)
- Gradient-based heatmaps

Deployment:
- Language: Python 3.8+
- Environment: Virtual environment (venv)
- Platforms: Windows, Linux, macOS

SECTION 2: CHALLENGES & SOLUTIONS
Challenge 1: Large Dataset Handling
- Problem: 61,955 images too large for memory
- Solution: Batch processing, data generators, stratified sampling

Challenge 2: Model Overfitting
- Problem: Training accuracy 95%, validation 87%
- Solution: Dropout layers, data augmentation, early stopping

Challenge 3: SHAP Compatibility
- Problem: SHAP TreeExplainer doesn't work with AdaBoost
- Solution: Used KernelExplainer (model-agnostic approach)

Challenge 4: Feature Scaling Issues
- Problem: Different feature ranges (N: 0-100, Temp: 10-40)
- Solution: StandardScaler for all numeric features

Challenge 5: Crop Mapping
- Problem: CNN classes don't match AdaBoost crop names
- Solution: Created mapping dictionary (CNN class → Crop name)

VISUAL ELEMENTS:
- Technology stack diagram showing all components
- Table showing challenges and solutions
- Icons for each technology
- Color-code challenges (red) and solutions (green)

SPEAKER NOTES:
"We used TensorFlow and Keras for the CNN model, Scikit-learn for the AdaBoost model, and SHAP for explainability. The frontend is built with Streamlit, which makes it easy to create an interactive web interface. We faced several challenges during development. The biggest was handling 61,955 images, which we solved using batch processing and data generators. We also had to prevent overfitting, which we solved using dropout, augmentation, and early stopping. Another challenge was making SHAP work with AdaBoost, which we solved by using KernelExplainer instead of TreeExplainer."
